Kaggle Ames House Prices¶

This project aims to create some regression models to predict the sale price of each home in Ames, Lowa. The dataset was collected from Kaggle competition website (https://www.kaggle.com/c/house-prices-advanced-regression-techniques), which includes 1460 observations and 81 variables that cover all aspects of residential homes. The complete code can be found here.

def fill_missing_data(frame, column, value = 'None', re_column='None'):
    filtered = frame[column].isnull()
    if re_column == 'None':
        frame.loc[filtered, column] = value
    else:
        frame.loc[filtered, column] = frame.loc[filtered, re_column]

4. Log transform¶

Transforming the skewed numeric features by taking log(feature + 1) will make the features more normal.

Some independent variables look like good candidates for log transformation, such asTotalBsmtSF, KitchenAbvGr, LotFrontage, LotArea and others. While ganining on regression transformation will smooth out some irregularities which could be important, such as 2ndFlrSF. These irregularities are good candidates for feature construction.

In [34]:
# log the attributes that follow the normal distribute
log_columns = ["LotFrontage", "LotArea","BsmtUnfSF",
               "TotalBsmtSF", "GrLivArea", "TotalArea", "SalePrice"]

skewed = skewed[(skewed > 0.8) | (skewed < -0.8)]
skewed = skewed.index.tolist()

pd.options.mode.chained_assignment = None 
df_data.loc[:,skewed] = np.log1p(df_data[skewed].values)

4. Normalization¶

An important concern with this dataset is that the input attributes all vary in their scales because they measure different quantities. It is almost always good practice to normalized the data before buliding model.

Normalization cannot be done before the splitting as we don't want to fit the MinMaxScaler on some observations that will later be used in the test set.

In [35]:
# split the dataset into dataset 80% and test 20%
